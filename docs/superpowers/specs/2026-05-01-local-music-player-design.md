# 本地简易音乐播放器设计（Windows 11）

## 1. 目标与范围

### 1.1 产品目标
- 打包为可双击即用的 Windows 11 `exe`。
- 在保证界面质感的同时，优先控制安装体积与运行内存。
- UI 风格接近苹果系统常见玻璃拟态视觉。

### 1.2 首版范围
- 本地文件夹扫描导入（不含在线音源）。
- 基础播放控制：播放/暂停、上一首/下一首、进度拖动、音量。
- 播放列表管理：创建、重命名、删除、增删曲目。
- 播放模式：顺序、单曲循环、列表循环、随机。
- 最近播放记录。
- 歌词显示（本地 `.lrc` 优先）。
- 全局快捷键（Windows 11）。
- 主题切换（浅色/深色玻璃风）。

### 1.3 非目标（首版不做）
- 在线音乐、账户系统、云同步。
- 复杂音效处理（均衡器、DSP）。
- 跨平台兼容（仅 Windows 11）。

## 2. 技术方案

### 2.1 推荐方案
采用 **Tauri + Vite + TypeScript + Howler/WebAudio**。

### 2.2 选择理由
- 相对 Electron：安装体积与内存占用通常更低。
- 相对纯原生 WPF：前端 UI 迭代速度更快，玻璃风实现更灵活。
- 符合“轻便 + 质感”平衡目标。

### 2.3 运行结构
- 前端：负责界面渲染、交互、状态管理。
- Tauri（Rust）后端：负责目录扫描、元数据读取、路径安全校验、设置持久化。
- 前后端通过 Tauri command 通信，前端不直接访问磁盘。

## 3. 架构与模块设计

### 3.1 模块划分
1. `library`（曲库）
   - 扫描本地目录并过滤支持格式：`mp3/flac/wav/m4a`。
   - 生成曲目索引：`id/path/title/artist/album/duration/cover/lrcPath`。

2. `player`（播放核心）
   - 管理当前曲目、队列、状态、进度、音量、播放模式。
   - 提供统一控制接口：`play/pause/seek/next/prev/setMode`。

3. `playlist`（播放列表）
   - 列表创建/重命名/删除。
   - 列表中曲目增删，引用曲库项而非复制文件。

4. `history`（最近播放）
   - 记录最近 N 首（建议 100），去重并更新时间。

5. `lyrics`（歌词）
   - 读取同名 `.lrc`，解析时间轴并根据播放进度高亮。

6. `settings`（设置）
   - 持久化主题、音量记忆、最近扫描目录、快捷键配置。

7. `ui-shell`（界面壳）
   - 侧栏（曲库/播放列表/最近）+ 主内容区 + 底部播放控制条。

### 3.2 数据流
1. 启动应用 -> 读取 `settings` -> 恢复主题和偏好。
2. 用户选择目录 -> `library.scan()` -> 分批写入索引 -> UI 增量渲染。
3. 用户点击曲目 -> `player.load(track)` -> `player.play()` -> `lyrics.sync(time)`。
4. 切歌事件 -> `history.push(track)` -> UI 更新当前播放信息。
5. 用户改主题/快捷键 -> `settings.save()`。

### 3.3 数据标识
- 曲目 `id` 使用规范化路径哈希生成，减少重复导入与路径变化带来的冲突。

## 4. UI/UX 设计方向（Windows 11）

### 4.1 视觉语言
- 半透明磨砂背景层。
- 卡片轻高光 + 细边框 + 柔和阴影。
- 动效短、平滑、不过度。
- 控制栏固定底部，确保常用操作可达。

### 4.2 主题
- 浅色玻璃主题。
- 深色玻璃主题。
- 两者共享同一组件体系，仅替换颜色与透明度参数。

### 4.3 可用性
- 曲目列表支持键盘导航。
- 播放状态和当前曲目信息在任意视图可见。
- 无歌词、无封面、文件异常时均有明确空态/提示。

## 5. 异常处理

### 5.1 扫描阶段
- 无权限目录：跳过并在结果中提示“部分目录不可访问”。
- 损坏或无法解析音频：标记不可播放，不中断扫描。
- 超大目录：显示分批进度并支持取消。

### 5.2 播放阶段
- 文件被移动或删除：当前项标记异常并自动尝试下一首。
- 解码失败：轻提示并记录错误计数。
- 输出设备变化：保留播放上下文，必要时重建音频上下文。

### 5.3 歌词阶段
- 缺失 `.lrc`：显示空状态，不视为错误。
- `.lrc` 格式异常：跳过坏行，解析可用行。

## 6. 性能与资源策略

### 6.1 扫描性能
- 文件遍历与元数据解析解耦：先展示基础列表，再异步补全时长/封面。
- 基于目录签名与修改时间的增量扫描缓存。

### 6.2 渲染性能
- 长列表虚拟滚动。
- 封面懒加载与缩略图缓存。
- 搜索/排序防抖（150-250ms）。

### 6.3 内存控制
- 不常驻原图封面，仅保留缩略图与必要缓存。
- 播放器仅持有当前曲目与少量预读，不批量实例化音频对象。

### 6.4 打包与启动
- 使用 Tauri release 构建并关闭不必要插件。
- 前端资源压缩分包。
- 首屏先渲染壳层与控制栏，曲库列表延后填充。

## 7. 测试与验收标准

### 7.1 功能验收
- 可扫描并导入本地目录。
- 支持 `mp3/flac/wav` 正常播放。
- 播放控制与播放模式行为正确。
- 播放列表增删改可用。
- 最近播放按时间正确记录。
- 有 `.lrc` 时歌词可随进度高亮。
- 主题切换和全局快捷键可用。

### 7.2 性能验收（首版目标）
- 冷启动到可交互 <= 2 秒（中等配置 Win11）。
- 扫描 3000 首歌曲时 UI 不冻结。
- 空闲常驻内存目标约 120-180MB（随封面与歌词数量浮动）。
- 连续播放 1 小时无明显持续性内存增长。

### 7.3 稳定性验收
- 损坏文件、缺失文件、无歌词文件不会导致崩溃。
- 扫描取消、切换目录、重复扫描不会破坏索引一致性。

## 8. 实施边界与后续

### 8.1 本设计适用边界
- 仅适用于 Windows 11 本地播放器首版。
- 实现阶段遵循“先可用、后增强”，不引入首版范围外功能。
